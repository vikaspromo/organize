# Email Corpus ‚Äî !organize Label

Local-only store of emails tagged with the `!organize` Gmail label. This is your working corpus for processing and analyzing these emails.

## What's here vs. what's tracked
This directory is **default-deny gitignored** (see `.gitignore`). Only the tooling code and documentation are tracked; all data stays on this machine and never reaches the remote.

| File | Tracked? | What |
|---|---|---|
| `parse-corpus.mjs` | yes (code) | classifies + indexes the raw messages |
| `sync-corpus.mjs` | yes (code) | fetches new mail incrementally, appends to the store |
| `auth.mjs` | yes (code) | one-time OAuth authorization |
| `README.md`, `.gitignore`, `SETUP.md` | yes | documentation + ignore rules |
| `store/` (raw RFC822 messages) | **no ‚Äî local only** | the email data of record |
| `index/corpus-index.json` + `.md` | **no ‚Äî local only** | derived index (metadata + gists) |
| `credentials.json` | **no ‚Äî local only** | OAuth client secret (download from Google Cloud) |
| `token.json` | **no ‚Äî local only** | refresh token (generated by auth.mjs) |
| `.sync-state.json` | **no ‚Äî local only** | last-sync metadata + counts |

## Scope (privilege-safe)
Defined by a Gmail label in your account, not hand-maintained or query-based:
- **Label:** `!organize` ‚Äî all emails tagged with this label
- **Includes:** Deleted emails, spam (in:anywhere) ‚Äî so nothing is silently absent from the record
- **Excludes:** Emails you haven't explicitly tagged

The label is the source of truth. Add/remove emails from the label in Gmail; `sync-corpus.mjs` will keep the corpus in sync.

## Pipeline
1. **`sync-corpus.mjs`** ‚Äî first run backfills everything tagged `!organize`; later runs fetch only mail newer than `.sync-state.json`. Dedupes by Gmail message id, retries transient network errors. Local only; read-only Gmail (OAuth, refresh token in `token.json`). Run on demand: `node corpus/sync-corpus.mjs`.
2. **`parse-corpus.mjs`** ‚Äî classifies every message and writes the index:
   - `substantive` ‚Äî regular email content ‚Üí **the main analysis set**
   - `calendar` ‚Äî Google Calendar auto-notifications ‚Üí parked
   - `autoreply` ‚Äî out-of-office / auto-responders ‚Üí parked
   - Reports counts of what was parked (no silent drops).
3. The index (`corpus-index.json` + `corpus-index.md`) is your data of record for analysis/processing.

## Quick Start
```bash
# One-time setup (SETUP.md has full details)
node corpus/auth.mjs

# First sync (pull all !organize emails)
node corpus/sync-corpus.mjs

# Index and classify
node corpus/parse-corpus.mjs

# View the index
cat corpus/index/corpus-index.md
```

## Workflow
- **Tag emails in Gmail** with `!organize` as you gather the ones you need to process
- **Sync periodically:** `node corpus/sync-corpus.mjs` (incremental ‚Äî only new emails)
- **Reindex:** `node corpus/parse-corpus.mjs` (regenerates the index after sync)
- **Process:** Use the `corpus-index.json` or browse `store/*.eml` files directly

## Health
`sync-corpus.mjs` records last-successful-sync time; the `.sync-state.json` file shows when the corpus was last updated. If syncs are failing, you'll see error messages ‚Äî re-run to retry (transient network errors are retried automatically).
